Freedom Project: Soccer Game

Context

The "Freedom Project" for SEP11 is a year-long project all about making something using JavaScript along with a third-party JS tool.

Content

For this project, I chose to make a soccer game because I enjoy watching and playing soccer. As my third-party JS tool, I used p5play which is "an introduction to Object Oriented Programming". My game has two players, the red player moves with WASD and shoots with "x" while the blue player moves with the Arrow Keys and shoots with "l". It also has a timer which has a limit of 5 minutes and a score with a limit of 3 goals.

Reflection

Challenges

A challenge I faced while making my project was getting to timer to work. For the timer, I wanted it to count up to 5 minutes. While I was able to make it count up, it would continue displaying the amount of seconds that had past rather than the minutes and seconds. After a few attempts, I looked at my notes and remembered about mod (%) which gives the remainder of the division of two numbers. With this, I was able to make the timer run correctly up to 4 minutes and displayed the timer with text('4:' + timerValue %60, width/2 -300, height/ 10). Another challenge I faced was getting the score to work. Whenever a player would score a goal, I would need to give that player a point and I wanted to display the score on the screen. At first I created a score variable for each player and when the condition that the ball was inside the net was true, I would run score1++ or score2++. However I didn't yet know how to make it display on the screen and this code would increase the score to infinity. After reviewing the p5play website, I found text() which allows you to show text on the screen. This was important for me to add the timer and the score. Not only this, but I figured I needed to add a reset function to my game so when there is a goal the game resets and the score is immediately updated. The reset function helped with the issue of the scores going to infinity and the text helped me display the updated score. Below is the code that makes the score work with comments that explain them.


                if(ball.x < 250 && ball.x > 200 && ball.y > 300 && ball.y < 500) { // check for player2 scoring
                    score2++; // increase score
                    reset(); // reset positions
                } else if(ball.x > 1550 && ball.x < 1600 && ball.y > 300 && ball.y < 500) { // check for player1 scoring
                    score1++; // increase score
                    reset(); // reset positions
                }
                text(score1 + " - " + score2, width/2 + 300, height/10) // display the updated score

Takeaways

  • Time management is vital in order to complete everything you need to do. I was able to complete all my tasks and my freedom project but I didn't make my game very complex and as good as I wanted to make it because I didn't manage my time correctly.
  • It is important to spend time on your tool. Learning your tool is a major part of the entire freedom project and since I spent a good amount of time on it I felt that it was easier for me to make my game though there are always some challenges.
  • Notes are very useful. Taking notes throughout the year was very helpful since I could go back to them at any time and figure out solutions to problems.

Next Steps

A next step is to dedicate more time to the freedom project like making my main project idea and learning my tool better. When we first chose our project idea and tool, I had a lot of hope that I would actually do better this year than last year and had a lot of ideas for my game to be very complex. However, I struggled to correctly manage my time so I wasn't really able to make the game I wanted so I plan on focusing more next year.
